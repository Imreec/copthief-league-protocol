# Cop–Thief League Protocol

**A shared wire protocol + self-serve conformance kit for cross-team Cop–Thief games**
(final project, Orchestration of AI Agents 203.3763, University of Haifa).

**Status: v0.1-draft** — written before the official final-project assignment publishes, so game
rules (board, scent, scoring, report schema) are placeholders. What this draft pins is the part
that is rule-agnostic and already proven to work between two independent implementations: message
framing, hashing, seeds, sequencing, and result settlement. **v1.0 lands after the assignment PDF.**

## Why this exists

In EX06, the inter-group bonus required two teams to agree byte-for-byte on hash constructions,
wire format, coordinate frames, and report canonicalization — and every agreement was negotiated
pairwise, by hand, hours before (and during) the live game. It worked, but it took two people
debugging live to get there. The final project makes cross-team games the *main* requirement, in a
league of ~6–8 teams — that's 15–28 pairings, and pairwise hand-negotiation does not scale.

This repo replaces pairwise negotiation with a published standard:

- **You implement to the spec once**, alone, on your own schedule.
- **You certify yourself** against machine-generated test vectors — no partner needed.
- Any two certified teams can then play with zero prior coordination beyond a match card.

## What's here

| File | What it is |
|---|---|
| [`SPEC.md`](SPEC.md) | The protocol: canonical JSON, transport, match setup, wire format, hash constructions, disclosure modes, failure handling, settlement |
| [`vectors/`](vectors/) | Machine-generated test fixtures for every hash construction (30 vectors) |
| [`verify_vectors.py`](verify_vectors.py) | Stdlib-only reference checker — `python verify_vectors.py` |

The vectors were generated by the EX06 implementation that played a live, hash-confirmed
cross-team series (both teams reached an identical report SHA-256 before emailing). They are not
aspirational — two independent codebases have already reproduced them.

## How to adopt (once you have a codebase)

1. Read `SPEC.md` — it's short. Sections marked **PROPOSED** are up for league discussion;
   everything else was live-tested in EX06.
2. Implement the four constructions in §2 + §6 (canonical JSON, position commit, state hash,
   seed→starts). In Python each is ~5 lines; `verify_vectors.py` contains reference code.
3. Point your test suite at `vectors/*.json`. When your implementation reproduces every fixture,
   your bytes match every other conformant team's bytes.
4. Implement the wire format (§5), handshake (§4.3), and settlement flow (§9).

## Giving feedback

This is a draft **so that it can be argued with** — especially the PROPOSED parts: the fenced-JSON
trailer, the match card, the joint-seed coin flip, delayed-reveal scent (Mode B), the
report-mismatch escalation, and the optional league services (lobby/registry, desync notary,
sparring server — SPEC Appendix B). Open an issue / message us. The goal is that when the assignment PDF
drops, the league only needs to fill in the rule values and bump to v1.0 — not design a protocol
under deadline pressure.
